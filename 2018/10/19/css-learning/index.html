CSS选择器

简单选择器

  • 类型（元素）选择器 h1,p{}
  • 类选择器 .class
  • ID选择器 #ID
  • 通用选择器 *{}

属性选择器

  • 存在和值(Presence and value)属性选择器
    • [attr]:选择包含attr属性的所有元素
    • [attr=val]:仅选择attr属性被赋值为val的所有元素
    • [attr~=val]:var值是attr值包含的被空格分隔的取值列表里的一个
  • 子串值（Substring value)属性选择器
    • [attr|=val]：attr属性的值是val或值以var-开头的元素
    • [attr^=val]：attr属性的值以var开头（包括var）的元素
    • [attr$=val]：attr属性的值以var结尾（包括var）的元素
    • [attr*=val]：attr属性的值中包含子字符串var的元素

伪选择器

  • 伪类 :Pseudo-class;当你希望样式在特定状态下才被呈现到指定元素时，可以往元素的选择器后面加上对应的伪类。例如当鼠标悬停在元素上面，或当一个复选框被禁用或被勾选，又或当一个元素是它在DOM树中父元素的第一个子元素时。
  • 伪元素 ::Pseudo-element;
    • ::after
    • ::before
    • ::first-letter
    • ::first-line
    • ::selection
    • ::backdrop

组合器和多个选择器

Combinations Select
A,B 匹配满足A或B的任意元素
A B 匹配条件：B是A的子节点
A>B 匹配条件：B是A的直接子节点
A+B 匹配条件：B是A的下一个的兄弟节点
A~B 匹配条件：B是A之后的兄弟节点的任意一个

层叠和继承

什么选择器在层叠中胜出取决于三个因素（前一种否决后一种）：

  1. 重要性(Importance):!important 重要性不影响就近性
  2. 专用性(Specificity)：id专用性比class更高,class比element专用性高

  3. 源代码次序(Source order)：后面的规则覆盖较早的规则

    关于规则混记的注记
    当多个CSS规则匹配相同的元素时，它们都会被应用到该元素中。只有在这之后，任何相互冲突的属性才会被评估。

控制继承

CSS为处理继承提供了四种特殊的通用属性值：

  • inherit：属性值设置与父元素一样
  • initial：设置为浏览器默认样式表中的值
  • unset：重制为自然值，如果属性自然继承，就表现的像inherit，否则像initial
  • revert：属性值被设置成自定义样式所定义的属性，否则属性值被设置成用户代理的默认样式

CSS速写属性all能够被应用到每一个继承属性，一次性应用所有继承属性。这里的值可以时继承属性里的任何一个(inherit,initial,unset,or revert)对于撤销对样式的修改，这是非常方便的一种方式

能继承的：text- 、line- 、font- 、color 、cursor

不能的：盒模型的、浮动的、定位的、background-color

  • box-sizing:boder-box; boder,padding包含在width,height内，实现响应式布局

行内元素

不能设置宽高度

脱离标准文档流

标准文档流限制多，要实现既并排又设宽高就要脱离标准流

脱标后没有行、块之分，都有行块二象性

1. 浮动

  • 浮动元素互相贴靠
  • 浮动元素有“自围”效果

  • 浮动清除（四种办法）

    1. 浮动元素的祖先元素加高度

      浮动在有高度的盒子中，就不会影响后面的浮动元素。但是工作上，绝对不会给所有的盒子加高度，因为麻烦且不能适应页面的快速变化

    2. clear:both;表示自己的内部元素，不受其他盒子的影响。但是导致margin失效

    3. 隔墙法

    在两部分浮动元素中间建墙，隔开两部分浮动，让后面的浮动元素，不去追前面的浮动元素。而内墙法不仅能够让后面的p不去追前面的p了，还能把第一个div撑出高度。这样，这个div的背景、边框就能够根据p的高度来撑开了

    1. overflow:hidden;溢出隐藏，所有溢出边框的内容，都隐藏掉

    只要给父亲加上overflow:hidden; 那么，父亲就能被儿子撑出高了，这是一个偏方

2. margin

  • margin的塌陷现象
    标准文档流中，竖直方向的margin不叠加，以较大的为准；脱离标准文档流中，竖直方向的margin叠加
  • 盒子居中margin:0 auto;
  1. 使用margin:0 auto;的盒子，必须有width
  2. 只有标准流的盒子，才能使用margin:0 auto;居中
  • 善用父亲的padding,而不是儿子的margin
  • margin这个属性，本质上描述的是兄弟和兄弟之间的距离； 最好不要用这个marign表达父子之间的距离

Flexbox

Why Flexbox?

  • Verticlally centering a block of content inside its parent.
  • Making all the children of a container take up an equal amount of the available width/height,regardless of how much width/height is available.
  • Making all columns in a multiple column layout adopt the same height even if they contain a different amount of content.

Flexbox defines how Flex Items are laid out inside of Flex Containers.

Flex Lines

Flex Items are positioned inside a Flex Container along a Flex Line.By default there is only one Flex Line per Flex Container.

Writing Modes

An important part of customizing your Flexbox is changing the direction of the flex line.Writing Modes are a new way to write text right-to-left,or even vertically,as you might in certain languages.direction:rtl;

The main Axis and the Cross Axis

To abstract over the writing-mode,Flexbox uses the concepts of the Main Axis and the Cross Axis.Flex Lines follow the Main Axis.The Cross Axis is perpendicular to the Main Axis.
It is crucial to understand the Main and Cross terminology before continuing.Everything in Flexbox is relative to these axes.

Properties of the Flex Container

flex-direcion

The default value of flex-direction is row.With this value,Flex Items are laid out in the direction of the writing-mode.Again,that means left-to-right,top-to-bottom by default.The outher values are as follows:

  • row-reverse: The Main Start and Main End are swapped.If the writing-mode is left to right,Flex Items are now laid out right to left.
  • column: The Main Axis and the Cross Axis are swapped.If the writing system is horizontal,the Flex Items are now laid out vertically.
  • column-reverse: Same as column,but reversed.

justify-content

The justify-content property of Flex Containers adjusts the positions of Flex Items on the Main Axis.
The poosible values are:

  • flex-start(default)
  • flex-end
  • center
  • space-between
  • space-around

align-items

align-items is complementary to justify-content.align-items adjusts the way Flex Items are positioned on the Cross Axis. The possible values are:

  • flex-start(default)
  • flex-end
  • center
  • baseline baseline causes Flex Items to be aligned along their baseline.
  • stretch

flex-wrap自动换行

Using flex-wrap you can create Flex Containers with mutiple FLex Lines.The possible values for flex-wrap are:

  • nowrap(default)
  • wrap If flex-wrap is set to wrap,Flex Items wrap onto additional Flex Lines if there is not enough room for them on one Flex Line.Additional Flex Lines are added in the direction of the Cross Axis.
  • wrap-reverse wrap-reverse is the same as wrap except new Flex Lines wil be added in the opposite direction on the Cross Axis.

align-content

align-content modifies the behavior of flex-wrap.It is similar to align-items,but instead of aligning Flex Items,it aligns Flex Lines.As you might expect,its possivle values are similar:

  • stretch(default)
  • flex-start
  • flex-end
  • center
  • space-between
  • space-around

flex-flow

flex-flow is a shorthand for flex-direction and flex-wrap 

 .flex-container{  
 -webkit-flex-flow:column nowrap;
 flex-flow: column nowrap;
}

Properties of Flex Items

A Flex Item is any direct child of a Flex Container.Flex Items are said to have a Main Size and a Cross Size.The Main Size is the size of the Flex Itemm in the dimention of the Main Axis.The Cross Size is the size of the Flex Item in the dimention of the Cross Axis.
The following properties adjust the behavior of Flex Items.

order

order is the simplest one setting orders for Flex Items adjusts their order when rendered.In this example,we set the order of one Flex Item to -1,causing it to be displayed before any other Flex Items.If you have four items with order values of 2,1,1,and 0 set on them respectively,their display order would be 4th,2nd,3rd,then 1st.

margin

Here we declare margin-right:auto; on the first Flex Item,causing all the extra space to be absorbed to the right of that element.

align-self

The align-self property of Flex Items effectively override the Flex Container’s align-items property for that item.It has the same possible values as align-items:

  • stretch(default)
  • flex-start
  • flex-end
  • center
  • baseline

flex

flex specifies how a Flex Item will be prioritized when free space is being distributed on the Main Axis.

flex:[number]

This syntax specifies a number that represents the radio of free space that should be taken by this Flex Item.It can be useful to set the number to 1 for every Flex Item,causing free space to be distributed evenly.

flex:initial

A Flex Item with its flex value set to initial will be inflexible when there is free space,but can shrink smaller if needed.

flex:auto

A Flex Item with its flex value set to auto will be fully flexible along the Main Axis.

flex:none

A Flex Item with its flex value set to none will be fully inflexible in all situations.

advanced flex

flex can also be used as a shorthand for specifying flex-grow,flex-shrink,and flex-basis in one declaration:

flex:[flex-grow][flex-shrink][flex-basis]

visibility

When it is implemented,visibility:collapse; will be distinct from visibility:hidden: and display:none; for Flex Items.With collapse,the element will effect the Cross Size of the Flex container,but it will not be diaplayed nor will it consume any space on the Main Axis.This will be useful for dynamically adding and removing Flex Items without affexting the Flex Container’s Cross Size.

  • sans-serif 无衬线字体 “sans”是法语单词“没有”

    垂直居中

    单行内联(inline-)元素垂直居中:通过设置内联元素高度和行高相等

    #v-box{
  height:120px;
  line-height:120px;
}

    多行元素垂直居中

  • 利用表布局（table）
    .center-table{
  display:table;
}
.v-cell{
  display:table-cell;
  vertical-align:middle;
}
  • 利用flex布局(flex)
    .center-flex{
  display:flex;
  flex-direction:column;
  justify-content:center;
}
  • 利用精灵元素(ghost element)，即在父容器内放一个100%高度的伪元素，让文本和伪元素垂直对齐，从而达到垂直居中的目的
    .ghost-center{
  position:relative;
}    
.ghost-center::before{
  content:"";
  display:inline-block;
  height:100%;
  width:1%;
  vertical-align:middle;
}
.ghost-center p{
  display:inline-block;
  vertical-align:middle;
  width:20rem;
}

    块级元素垂直居中

  • 固定高度的块级元素
    .parent{
  position:relative;
}
.child{
  position:absolute;
  top:50%;
  height:100px;
  margin-top:-50px;
}
  • 未知高度的块级元素
    .parent{
  position:relative;
}
.child{
  position:absolute;
  top:50%;
  transform:translateY(-50%);//向左Y轴移动自身长宽的50%
}

    水平垂直居中

    固定宽高元素水平垂直居中：通过margin平移元素整体宽度的一半，使元素水平垂直居中

    .parent{
  position:relative;
}
.child{
  width:300px;
  height:100px;
  padding:20px;
  position:absolute;
  top:50%;
  left:50%;
  margin:-70px 0 0 -170px;
}

    未知宽高元素水平垂直居中：利用2D变换，在水平和垂直两个方向平移宽高的一半，从而使元素水平垂直居中

      position:relative;
}
.child{
  position:absolute;
  top:50%;
  left:50%;
  transform:translate(-50%,-50%);
}

    利用flex布局,其中justify-content用于设置或检索弹性盒子元素在主轴（横轴）方向上的对齐方式；而align-items属性定义flex子项在flex容器的当前行的侧轴（纵轴）方向上的对齐方式

    .parent{
  display:flex;
  justify-content:center;
  align-items:center;
}

    利用grid布局：兼容性较差

    .parent{
  height:140px;
  display:grid;
}
.child{
  margin:auto;
}
